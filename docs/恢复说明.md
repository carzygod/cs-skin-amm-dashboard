# 恢复说明

## 1. 恢复目标

本项目用于恢复并整理 CS2 饰品跨平台库存对敲套利系统的需求、开发方案、数据采集方案和可运行原型。恢复后的重点是：

- 明确即时求购利差和理论挂单利差的区别。
- 明确费用、充值成本、库存、余额和流动性过滤规则。
- 提供一个可运行的利差机会中心。
- 提供国内外平台目录、交易所列表和连通性探测。
- 保持当前版本为执行预检模式，不连接真实平台，不执行真实交易。

## 3. 原型能力

当前原型可以：

- 加载样例平台、费率、库存、余额和盘口数据。
- 计算多平台参考价和公允价。
- 计算 A 侧求购订单 bid VWAP。
- 计算 B 侧现货卖盘 ask VWAP。
- 估算 maker 买入目标价和理论挂单目标价。
- 扣除平台费率、充值成本、汇率成本和风险缓冲。
- 输出即时求购利差和理论挂单利差。
- 对流动性、库存和余额不足的机会进行剔除或降级。
- 展示机会列表、报价矩阵和执行预检预案。
- 展示 CSGOSKINS.GG markets 目录中的 32 个国内外平台。
- 展示每个平台的连通性、延迟、HTTP 状态、费率状态和真实行情接入状态。

当前原型不会：

- 连接真实账号。
- 读取浏览器会话。
- 读取或保存令牌。
- 执行真实买入、卖出、挂单、撤单、充值或提现。
- 触发 Steam Mobile 令牌确认。
