# 验收复查

## 1. 复查范围

本复查用于确认当前原型是否符合已整理的产品需求、开发文档和平台接入矩阵。复查重点是：

| 范围 | 复查目标 |
| --- | --- |
| Dashboard 结构 | 是否已经改为左侧 tab 的常规 dashboard |
| 交易所列表 | 是否展示当前支持平台、连通性、延迟和接入状态 |
| 平台覆盖 | 是否包含国内外平台，而不是只展示国内平台 |
| 数据边界 | 是否明确区分目录数据、连通性数据、样例盘口和真实可交易数据 |
| 标的范围 | 是否已删除皮肤样例，只保留武器箱、贴纸、印花胶囊和签名胶囊 |
| 机会模型 | 是否拆分即时求购利差和理论挂单利差 |
| 费用与流动性 | 是否同时计算毛利差、净利差、充值成本、风险缓冲，并过滤流动性不足的机会 |
| 安全边界 | 当前原型是否不连接真实账号、不读取敏感信息、不执行真实交易 |

## 2. 当前结论

| 项目 | 结论 | 说明 |
| --- | --- | --- |
| 左侧 tab dashboard | 符合 | 页面包含总览、利差机会、交易所列表、平台行情、操作预案五个主视图 |
| 交易所列表 | 符合 | 每个平台一行展示市场类型、目录统计、连通性、延迟、HTTP 状态、样例行情覆盖、费率状态和正式接口状态 |
| 国外平台覆盖 | 符合 | 平台目录纳入 CSGOSKINS.GG markets 的 32 个国内外市场，并手动新增 market.csgo.com |
| 数据完整表达 | 符合 | 已明确当前有目录统计、连通性探测和 33 平台样例盘口，正式平台 adapter 待接入 |
| 即时利差与理论利差 | 符合 | 机会类型使用 `INSTANT_BUY_ORDER` 和 `THEORETICAL_LISTING` 分开计算、展示和筛选 |
| 费用模型 | 符合 | 利润计算同时输出毛利润、净利润、毛利差、净利差，并包含卖出费、买入费、maker/taker、充值成本、汇率成本和风险缓冲 |
| 流动性规则 | 符合 | 即时机会必须检查 bid 深度、ask 深度、库存、余额和流动性评分 |
| 标的范围 | 符合 | 当前样例标的只包含武器箱、贴纸、印花胶囊和签名胶囊，已移除皮肤样例 |
| 安全边界 | 符合 | 当前原型只做预检，不连接真实平台账户，不提交订单，不处理充值、支付或令牌 |

## 3. 关键验收项

| 验收项 | 验收方式 | 通过标准 |
| --- | --- | --- |
| 平台目录数量 | 检查 `/api/platforms` 或快照中的平台覆盖摘要 | 返回 33 个平台 |
| 样例行情覆盖 | 检查 `/api/snapshot` 中的报价矩阵 | 每个饰品返回 33 个平台报价 |
| 交易所列表数量 | 打开 dashboard 并切换交易所列表 | 页面渲染 33 行平台 |
| 连通性字段 | 检查 `/api/platform-connectivity` | 每个平台包含状态、延迟、HTTP 状态和探测目标 |
| 左侧导航 | 检查页面结构 | 使用侧边栏 tab 切换视图 |
| 移动端适配 | 检查手机宽度页面 | 不出现横向溢出，交易所列表可读 |
| 理论机会边界 | 检查机会输出 | 理论挂单机会不得标记为强执行 |
| 即时机会边界 | 检查机会输出 | 只有库存、余额、盘口和利润达标才可标记为强执行 |
| 机会筛选排序 | 检查利差机会页面 | 支持按毛/净利差排序，按最低/最高利差和买卖交易所过滤 |
| 文档一致性 | 检查需求、开发、平台矩阵和恢复说明 | 文档均体现交易所列表、平台覆盖和数据边界 |

## 4. 不符合项处理

复查过程中发现并处理了以下问题：

| 问题 | 处理结果 |
| --- | --- |
| 需求文档未明确交易所列表和左侧 tab dashboard | 已补充 Dashboard 需求和验收标准 |
| 文档未清晰区分目录数据与真实可交易数据 | 已补充平台数据边界说明 |
| 平台目录统计与当前目录来源不一致 | 已同步 33 个平台的目录统计 |
| PDF 文档合集落后于当前文档 | 已重新导出，并清理日期型元数据 |

## 5. 剩余边界

| 边界 | 状态 | 后续要求 |
| --- | --- | --- |
| 真实盘口 adapter | 未接入 | 逐平台接入 listings、buyOrders、recentSales |
| 真实费率 | 未验证 | 平台官方或账户侧确认后才能用于强执行 |
| 库存与余额 | 当前为样例数据 | 用户授权后接入只读同步 |
| 受控执行 | 未接入 | 必须在预检、审计和用户令牌确认边界内设计 |

当前原型符合“预检 dashboard”和“平台覆盖可视化”的要求，不应被描述为已经具备真实交易执行能力。
