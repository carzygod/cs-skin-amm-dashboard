# CS-AMM 开发文档

## 1. 工程目标

CS-AMM 的工程目标是把跨平台饰品套利拆成可验证的数据流水线：

```text
数据采集 -> 标准化 -> 价格预言机 -> 机会引擎 -> 流动性过滤 -> Dry Run -> 受控执行任务
```

当前原型实现到 Dry Run 层，只使用样例数据，不连接真实平台，不执行真实交易。

## 2. 架构原则

| 原则 | 说明 |
| --- | --- |
| 采集与执行分离 | 行情、库存、余额、费率读取不得和执行动作耦合 |
| 原始数据与标准化数据分离 | 原始 payload 用于审计，标准化结构用于计算 |
| 价格预言机前置 | 机会引擎不得直接消费平台展示价 |
| 两类机会强制拆分 | 即时求购利差与理论挂单利差使用不同口径 |
| 流动性先于利润 | 流动性不足时即使利润高也不采纳 |
| Dry Run 优先 | 任何真实执行前必须先能生成可审查预案 |
| 不保存敏感信息 | 不保存账号凭证、会话、支付信息或令牌 |

## 3. 当前原型模块

| 模块 | 职责 |
| --- | --- |
| 样例数据 | 提供平台、费率、库存、余额、盘口深度和交易速度 |
| 价格预言机 | 生成 referencePrice、fairPrice、平台级结构化报价 |
| 机会引擎 | 生成即时求购利差和理论挂单利差 |
| 费用引擎 | 计算卖出费、买入费、充值成本、汇率成本和风险缓冲 |
| 流动性引擎 | 计算 bid VWAP、ask VWAP、滑点、深度和综合评分 |
| Dry Run | 按机会类型生成执行前预案 |
| Dashboard | 展示指标、筛选、机会卡片、预言机矩阵和 Dry Run |

## 5. 价格预言机

价格预言机输出每个平台在同一饰品上的可解释报价，而不是单个平台展示价。

```ts
interface OracleResult {
  itemId: string
  referencePrice: number
  fairPrice: number
  quotes: PlatformQuote[]
  confidence: 'HIGH' | 'MEDIUM' | 'LOW' | 'INVALID'
}
```

```ts
interface PlatformQuote {
  platformId: string
  takerSellPrice: number | null
  takerSellQuantity: number
  takerSellSlippageRate: number
  spotBuyPrice: number | null
  spotBuyQuantity: number
  spotBuySlippageRate: number
  makerBuyTargetPrice: number | null
  makerBuyProbability: number
  theoreticalListingPrice: number | null
  listingFillProbability: number
  liquidityScore: number
  freshnessMs: number
  confidence: 'HIGH' | 'MEDIUM' | 'LOW' | 'INVALID'
}
```

### 5.1 referencePrice

```text
midPrice = (bestBid + bestAsk) / 2
referencePrice = 多平台 midPrice 的加权中位数
```

权重包含：

| 因子 | 作用 |
| --- | --- |
| 平台健康度 | 降低不稳定平台权重 |
| 数据新鲜度 | 降低过期数据权重 |
| 盘口深度 | 降低薄盘口权重 |

### 5.2 bid VWAP

用于即时求购利差的 A 侧卖出价格。

```text
takerSellPrice = 吃掉目标数量后 bids 的成交均价
```

如果 bid 深度无法覆盖目标数量，则该报价不得用于强执行。

### 5.3 ask VWAP

用于即时补货和理论入场成本。

```text
spotBuyPrice = 吃掉目标数量后 asks 的成交均价
```

如果 ask 深度无法覆盖目标数量，则该报价不得用于机会采纳。

### 5.4 theoreticalListingPrice

用于理论挂单利差。

```text
theoreticalListingPrice = 当前卖盘附近的目标挂单价
```

该价格必须结合成交速度、排队数量、价差和平台健康度估算 `listingFillProbability`。

## 6. 机会引擎

机会引擎枚举同一饰品在 A/B 平台之间的组合，并分别构建两种机会。

### 6.1 即时求购利差

```text
type = INSTANT_BUY_ORDER
sellMode = TAKER_BUY_ORDER
buyMode = TAKER_SPOT
```

计算步骤：

| 步骤 | 内容 |
| --- | --- |
| 1 | 读取 A 侧 bid VWAP |
| 2 | 读取 B 侧 ask VWAP |
| 3 | 校验 A 侧可卖库存 |
| 4 | 校验 B 侧可用余额 |
| 5 | 校验 A 侧求购深度和 B 侧现货深度 |
| 6 | 扣除卖出费、买入费、充值成本、汇率成本和风险缓冲 |
| 7 | 利润率和流动性达标时标记为强可执行 |

### 6.2 理论挂单利差

```text
type = THEORETICAL_LISTING
sellMode = MAKER_LISTING
buyMode = TAKER_SPOT
```

计算步骤：

| 步骤 | 内容 |
| --- | --- |
| 1 | 读取 A 侧目标挂单价 |
| 2 | 读取 B 侧 ask VWAP |
| 3 | 校验 A 侧可挂库存 |
| 4 | 校验 B 侧可用余额 |
| 5 | 估算挂单成交概率 |
| 6 | 扣除费用和挂单风险缓冲 |
| 7 | 只进入观察池，不标记为强可执行 |

## 7. Opportunity 输出

```ts
interface Opportunity {
  opportunityId: string
  type: 'INSTANT_BUY_ORDER' | 'THEORETICAL_LISTING'
  typeName: string
  executionClass: 'IMMEDIATE' | 'THEORETICAL'
  itemId: string
  sellPlatformId: string
  buyPlatformId: string
  sellMode: string
  buyMode: string
  quantity: number
  sellPrice: number
  buyPrice: number
  unitNetProfit: number
  totalNetProfit: number
  netProfitRate: number
  liquidityScore: number
  level: 'STRONG_EXECUTABLE' | 'WATCH_ONLY' | 'NEEDS_VERIFICATION' | 'REJECTED'
  executable: boolean
  feeBreakdown: FeeBreakdown
  liquidityCheck: LiquidityCheck
  rejectReasons: string[]
}
```

## 8. 费用计算

| 字段 | 说明 |
| --- | --- |
| sellPlatformFee | 卖家基础费用 |
| sellExecutionFee | maker/taker 卖出执行费用 |
| buyPlatformFee | 买家基础费用 |
| buyExecutionFee | maker/taker 买入执行费用 |
| depositCost | 充值或资金准备成本 |
| fxCost | 汇率或跨币种成本 |
| riskBuffer | 滑点、延迟、调价和异常缓冲 |

## 9. 流动性评分

综合评分由以下部分组成：

| 因子 | 说明 |
| --- | --- |
| sellScore | bid VWAP 是否可完整成交以及滑点大小 |
| buyScore | ask VWAP 是否可完整成交以及滑点大小 |
| modelScore | maker 买入概率或挂单成交概率 |
| freshnessScore | 盘口数据新鲜度 |
| platformHealthScore | 平台采集和执行稳定性 |

低于策略阈值的机会不得标记为强可执行。

## 13. 后续接入真实平台

真实平台接入建议顺序：

1. 只读行情。
2. 只读盘口。
3. 只读费率。
4. 只读库存。
5. 只读余额。
6. Dry Run 自动预检。
7. 用户授权后的受控任务执行。
8. Steam Mobile 人工确认闭环。
9. 对账和异常恢复。

每一步都应具备降级策略、审计日志和人工复核入口。
