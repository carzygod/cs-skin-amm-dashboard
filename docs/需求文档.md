# CS-AMM 需求文档

## 1. 文档目标

本文档定义 CS2 饰品跨平台库存对敲套利系统的产品需求。系统目标不是简单展示“低买高卖”的价差，而是基于多平台库存、余额、盘口深度、平台费率、充值成本、成交概率和人工令牌确认，识别可以被采纳的套利机会，并把机会清晰拆分为即时利差和理论利差。

当前原型只演示数据、计算、平台目录、连通性探测和 Dry Run，不连接真实平台账户，不执行真实交易。

## 2. 总体目标

| 目标 | 说明 | 验收标准 |
| --- | --- | --- |
| 实时发现利差 | 采集多平台同款饰品的买盘、卖盘、库存、余额、费率 | 输出按净利差、毛利差和可执行性排序的机会列表 |
| 区分两类利差 | 将“即时求购利差”和“理论挂单利差”分开计算、展示和筛选 | 页面、接口、文档均有明确类型字段 |
| 验证流动性 | 使用盘口 VWAP、可成交数量、成交速度、平台健康度进行过滤 | 流动性不足的机会不得进入可采纳列表 |
| 计算毛利差与净利差 | 毛利差用于观察原始价差，净利差扣除卖家费、买家费、maker/taker 费、充值成本、汇率成本、风险缓冲 | 机会卡片和接口必须同时输出毛利差、净利差、手续费估算和总成本率 |
| 管理库存 | 跟踪可用、预留、挂单、在途、冷却和异常库存 | 防止同一件库存被重复用于多个机会 |
| 管理余额 | 跟踪可用、预留、锁定、在途和异常余额 | 防止同一笔资金被重复占用 |
| 降低用户操作负担 | 用户只查看机会、发起任务、审查 Steam Mobile 令牌 | 手机端交互大按钮、少输入、少层级 |
| 保留最终人工审核 | 资产交割相关确认由用户在 Steam Mobile 中完成 | 系统不得绕过人工令牌确认 |
| 覆盖国内外平台目录 | 将国内外主流 CS2 饰品平台纳入统一目录和接入矩阵 | 平台列表、连通性和接入状态在 dashboard 中可见 |

## 3. 机会类型

### 3.1 即时求购利差

即时求购利差用于快速对敲，是系统的主交易候选。

```text
A 侧：卖出已有库存，吃已有求购订单
B 侧：从现货卖盘同步买入同款库存补回
```

| 项目 | 计算口径 |
| --- | --- |
| A 侧卖出价 | 吃掉目标数量后的 buy order / bid VWAP |
| B 侧买入价 | 吃掉目标数量后的 ask VWAP |
| 可执行数量 | min(A 侧可用库存, A 侧求购深度, B 侧现货深度, B 侧余额可覆盖数量) |
| 毛利润 | A 侧卖出毛额 - B 侧买入毛额 |
| 净利润 | A 侧卖出净到账 - B 侧买入总成本 - 风险缓冲 |
| 采纳条件 | 利润为正、利润率达标、盘口深度足够、数据新鲜、库存和余额足够 |
| 执行属性 | 可进入强可执行候选 |

### 3.2 理论挂单利差

理论挂单利差用于观察，不直接作为一键套利执行对象。

```text
B 侧：从现货卖盘买入或补货
A 侧：以目标价格挂单卖出
```

| 项目 | 计算口径 |
| --- | --- |
| A 侧目标卖价 | 当前最低卖盘附近的 maker listing 目标价 |
| B 侧买入价 | 吃掉目标数量后的 ask VWAP |
| 可观察数量 | min(A 侧可用库存, B 侧现货深度, B 侧余额可覆盖数量) |
| 理论毛利润 | A 侧目标挂单毛额 - B 侧买入毛额 |
| 理论净利润 | A 侧挂单成交后净到账 - B 侧买入总成本 - 挂单风险缓冲 |
| 主要风险 | 不一定成交、排队靠后、价格回撤、成交速度不足 |
| 执行属性 | 只进入观察池，不进入自动执行候选 |

### 3.3 两类利差对比

| 维度 | 即时求购利差 | 理论挂单利差 |
| --- | --- | --- |
| 卖出端 | 已存在的求购订单 | 用户挂出的卖单 |
| 成交确定性 | 高 | 不确定 |
| 核心数据 | bid VWAP、求购深度、库存 | ask 附近目标挂单、成交速度、排队位置 |
| 利润性质 | 即时毛利差与即时净利差 | 理论毛利差与理论净利差 |
| 是否适合一键套利 | 是，前提是通过所有预检 | 否，只用于观察和人工判断 |
| UI 默认优先级 | 最高 | 次级 |

## 4. 用户交互模式

| 场景 | 用户行为 | 系统行为 |
| --- | --- | --- |
| 打开实时利差页 | 查看机会列表 | 展示即时求购、理论挂单、强可执行、观察、利差区间和交易所筛选 |
| 查看机会卡片 | 检查利润、数量、费用、流动性 | 展示 A/B 平台路径、VWAP、毛利润、净利润、手续费估算、充值成本、风险缓冲 |
| 发起套利任务 | 点击“一键套利”或等价操作 | 后端重新预检，不直接使用旧快照 |
| 等待执行 | 用户不逐个平台手动操作 | 系统推进平台侧可自动化流程或 Dry Run |
| 审查令牌 | 用户在 Steam Mobile 中确认或拒绝 | 系统等待并同步最终状态 |
| 查看结果 | 查看盈亏、库存、余额、订单状态 | 系统完成对账并写入任务记录 |

| 排序区 | 净利差高到低、净利差低到高、毛利差高到低、毛利差低到高、净利润高到低、手续费低到高 | 排序仅影响当前视图，不改变原始快照数据 |

| 价格预言机 | 参考价、公允价、求购 VWAP、现货 VWAP、目标挂单、流动性 | 当前原型为 32 个目录平台展示结构化样例报价 |
| 平台覆盖 | CSGOSKINS.GG 市场目录、接入状态、平台统计 | 所有目录平台都必须展示，不得只展示已接入平台 |
| 交易所列表 | 当前支持平台、市场类型、目录统计、连通性、延迟、HTTP 状态、样例行情覆盖、费率状态、正式接口状态 | 每个平台都必须有一行，不得只展示国内平台 |
| Dry Run | 机会类型、执行步骤、预检提醒、边界说明 | 不提交真实订单 |

| --- | --- | --- |
| 平台目录 | 已纳入 CSGOSKINS.GG markets 目录中的 32 个市场 | 否，仅用于覆盖与接入管理 |
| 平台统计 | 展示评分、评论、物品数、报价数、市场价值、平均折扣、月访问量 | 否，仅用于接入优先级和平台概览 |
| 连通性探测 | 展示平台官网或目录页的在线、可访问、异常、超时、不可达和延迟 | 否，仅说明网络层状态 |

| 真实费率 | 待平台或账户侧确认 | 未确认时不得标为强执行 |
| 库存与余额 | 当前使用样例数据 | 真实只读接入后才可用于执行预检 |

## 6. 费用模型

| 成本项 | 含义 | 是否进入净利润 |
| --- | --- | --- |
| sellerFeeRate | 卖出平台收取的卖家费 | 是 |
| buyerFeeRate | 买入平台收取的买家费 | 是 |
| makerFeeRate | 挂单成交或 maker 行为费率 | 是 |
| takerFeeRate | 吃单成交或 taker 行为费率 | 是 |
| depositCostRate | 充值、入金、资金准备成本 | 是 |
| fxCostRate | 跨币种兑换或资金换汇成本 | 是 |
| riskBuffer | 滑点、延迟、调价、订单失败缓冲 | 是 |

即时求购利差公式：

```text
A侧卖出毛额 = A侧bid VWAP * 数量
B侧买入毛额 = B侧ask VWAP * 数量
毛利润 = A侧卖出毛额 - B侧买入毛额
毛利差 = 毛利润 / B侧买入毛额
A侧卖出净到账 = A侧bid VWAP * 数量 * (1 - 卖家费率 - taker费率)
B侧买入总成本 = B侧ask VWAP * 数量 * (1 + 买家费率 + taker费率 + 充值成本率 + 汇率成本率)
手续费估算 = 卖出费用 + 买入费用 + 充值成本 + 汇率成本
净利润 = A侧卖出净到账 - B侧买入总成本 - 风险缓冲
净利差 = 净利润 / B侧买入总成本
```

理论挂单利差公式：

```text
A侧挂单毛额 = A侧目标挂单价 * 数量
B侧买入毛额 = B侧ask VWAP * 数量
理论毛利润 = A侧挂单毛额 - B侧买入毛额
理论毛利差 = 理论毛利润 / B侧买入毛额
A侧挂单净到账 = A侧目标挂单价 * 数量 * (1 - 卖家费率 - maker费率)
B侧买入总成本 = B侧ask VWAP * 数量 * (1 + 买家费率 + taker费率 + 充值成本率 + 汇率成本率)
手续费估算 = 卖出费用 + 买入费用 + 充值成本 + 汇率成本
理论净利润 = A侧挂单净到账 - B侧买入总成本 - 挂单风险缓冲
理论净利差 = 理论净利润 / B侧买入总成本
```

## 7. 流动性规则

| 检查项 | 即时求购利差 | 理论挂单利差 |
| --- | --- | --- |
| A 侧求购深度 | 必须覆盖目标数量 | 可作为需求强度参考 |
| B 侧现货深度 | 必须覆盖目标数量 | 必须覆盖目标数量 |
| VWAP 滑点 | 超阈值剔除或降级 | 超阈值剔除或降级 |
| 数据新鲜度 | 过期不得强执行 | 过期只允许观察 |
| 成交概率 | 不适用或低权重 | 必须评估挂单成交概率 |
| 平台健康度 | 低分不得强执行 | 低分不得强执行 |

## 13. 验收标准

| 领域 | 标准 |
| --- | --- |
| 机会计算 | 明确拆分即时求购利差和理论挂单利差 |
| 费用计算 | 同时输出毛利差和净利差，净利润包含平台费用、maker/taker、充值成本、汇率成本和风险缓冲 |
| 流动性 | 盘口深度不足、余额不足、库存不足的机会不得被采纳 |
| 价格预言机 | 输出参考价、公允价、VWAP、目标挂单价、置信度和流动性评分 |
| Dashboard | 能筛选两类机会、按毛/净利差排序、按利差区间和交易所过滤，并展示费用拆解与 Dry Run |

| 平台覆盖 | 国内外平台统一纳入目录，不得把国外平台从 dashboard 隐藏 |
| 安全边界 | 当前原型不连接真实账号，不保存敏感信息，不执行真实交易 |
