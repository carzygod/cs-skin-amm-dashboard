# 平台接入矩阵

## 1. 目标

平台接入矩阵用于统一管理 CS-AMM 后续要覆盖的全部市场。平台列表以 CSGOSKINS.GG markets 页面为目录来源，系统必须把每个平台纳入同一套字段、同一套能力检查和同一套接入状态管理。

当前状态说明：

| 状态 | 含义 |
| --- | --- |
| 已纳入目录 | 平台已经进入系统平台目录，可在 dashboard 中展示 |
| 真实行情待接入 | 尚未接入该平台的真实盘口、成交、费率和账户数据 |
| 费用待验证 | 平台费率、充值成本、支付成本、汇率成本未经过官方或账户侧确认 |
| 执行待设计 | 尚未设计该平台的受控执行、异常恢复和对账流程 |

## 2. 必备字段

每个平台都必须具备以下字段，即使暂未接入真实接口，也必须显式标注状态，不允许留空。

| 字段 | 说明 |
| --- | --- |
| platformId | 系统内部唯一标识 |
| name | 平台展示名称 |
| marketType | 市场类型，如官方市场、现金市场、P2P 市场、交易机器人市场 |
| sourceUrl | CSGOSKINS.GG 对应市场页 |
| directoryStatus | 是否已纳入目录 |
| liveConnectionStatus | 真实行情接入状态 |
| feeProfileStatus | 费率确认状态 |
| dataCompletenessStatus | 数据完整性状态 |
| marketStats | 评分、评论数、物品数、报价数、市场价值、折扣、访问量 |
| requiredCapabilities | 必须补齐的能力列表 |
| capabilityStatus | 每项能力的当前状态 |

## 3. 必备能力

| 能力 | 说明 |
| --- | --- |
| publicMarketStats | 公开市场统计 |
| itemMapping | 饰品名称、磨损、模板、平台 itemId 映射 |
| listings | 卖盘/现货挂单 |
| buyOrders | 求购订单 |
| recentSales | 近期成交 |
| feeProfile | 买入费、卖出费、maker/taker 费 |
| depositCost | 充值、支付、入金、换汇成本 |
| inventoryRead | 用户库存只读同步 |
| balanceRead | 用户余额只读同步 |
| precheck | 执行前预检 |
| execution | 用户授权后的受控执行 |
| reconciliation | 订单、库存、余额和盈亏对账 |

## 5. 接入优先级

| 优先级 | 平台 | 原因 |
| --- | --- | --- |
| P0 | Steam、CSFloat、Skinport、BUFF163、DMarket、LIS-SKINS、CS.MONEY、Tradeit.gg | 市场规模、访问量、报价量和跨区域套利价值较高 |
| P1 | GameBoost、Skin.Land、white.market、SkinBaron、WAXPEER、ShadowPay、CS.TRADE | 有一定报价深度和成交观察价值 |
| P2 | 其余平台 | 先纳入目录，等只读行情稳定性和费用可验证后再推进 |

## 6. 验收标准

| 验收项 | 标准 |
| --- | --- |
| 平台数量 | dashboard 和 `/api/platforms` 返回 32 个 CSGOSKINS.GG 市场 |
| 字段完整 | 每个平台都有 sourceUrl、marketStats、requiredCapabilities、capabilityStatus |
| 状态真实 | 未接入平台必须显示待接入，不得伪装成已可交易 |
| 后续可扩展 | 新增平台适配器时只更新该平台 capabilityStatus 和 adapter 实现 |
