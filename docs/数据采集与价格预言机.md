# 数据采集与价格预言机设计

## 1. 核心观点

系统的核心不是“采到价格”，而是把原始价格加工成可执行、可解释、可拒绝的价格信号。平台展示价不能直接进入套利引擎，必须先经过标准化、深度计算、费用校验、流动性评分和置信度判定。

价格预言机的输出应包含：

```text
参考价 + 可执行价格 + 可执行数量 + 费用拆解 + 流动性评分 + 置信度 + 剔除原因
```

## 2. 数据采集分层

| 层级 | 数据 | 作用 |
| --- | --- | --- |
| L1 公开行情层 | 展示价、最高求购价、最低卖价、近期成交价 | 初筛、异常检测 |
| L2 盘口深度层 | 多档 bids、asks、数量、订单数 | 计算 bid VWAP、ask VWAP、深度和滑点 |
| L3 账户状态层 | 库存、余额、预留、锁定、冷却、异常 | 判断机会是否可执行 |
| L4 执行状态层 | 订单提交、部分成交、等待令牌、取消、失败 | 推进任务状态机 |
| L5 对账层 | 订单、库存、余额、实际盈亏 | 复盘、风控和异常恢复 |

## 3. 数据源优先级

| 优先级 | 来源 | 适用场景 | 风险 |
| --- | --- | --- | --- |
| P0 | 平台正式 API | 行情、盘口、订单、库存、余额 | 依赖平台开放能力 |
| P1 | 页面只读采集 | API 不完整时读取展示数据 | 页面结构可能变化 |
| P2 | 人工配置 | 费率、充值成本、阈值、平台启停 | 需要定期复核 |
| P3 | 历史缓存 | 短时断网或平台异常时降级展示 | 不得用于强执行 |
| 禁止 | 未验证展示价 | 任何强执行场景 | 容易误判 |

## 6. referencePrice

`referencePrice` 用于异常检测，不用于直接执行。

```text
midPrice = (bestBid + bestAsk) / 2
referencePrice = 多平台 midPrice 的加权中位数
```

权重因子：

| 因子 | 说明 |
| --- | --- |
| 平台健康度 | 平台采集和状态越稳定，权重越高 |
| 数据新鲜度 | 越新的盘口权重越高 |
| 盘口深度 | 越深的盘口权重越高 |
| 成交速度 | 最近成交越活跃，权重越高 |

## 7. bid VWAP

bid VWAP 用于即时求购利差的 A 侧卖出价格。

```text
takerSellPrice = sum(bidPrice_i * filledQuantity_i) / targetQuantity
```

计算要求：

| 检查 | 处理 |
| --- | --- |
| bid 深度覆盖目标数量 | 可继续计算 |
| bid 深度不足 | 不得进入强执行 |
| 吃单后滑点过大 | 降级或剔除 |
| 数据过期 | 降级或剔除 |

## 8. ask VWAP

ask VWAP 用于 B 侧现货买入成本。

```text
spotBuyPrice = sum(askPrice_i * filledQuantity_i) / targetQuantity
```

适用场景：

| 场景 | 是否使用 ask VWAP |
| --- | --- |
| 即时求购利差的 B 侧补货 | 是 |
| 理论挂单利差的 B 侧入场 | 是 |
| maker 买入目标价 | 否，maker 目标价单独估算 |

## 9. makerBuyTargetPrice

maker 买入目标价用于观察或策略辅助，不应混同于即时补货成本。

```text
makerBuyTargetPrice = min(bestBid + tickSize, bestAsk - tickSize, maxReasonableBuyPrice)
```

影响因素：

| 因素 | 作用 |
| --- | --- |
| 当前最高买单 | 判断是否需要排队 |
| 当前最低卖单 | 判断卖方接受概率 |
| tickSize | 控制挂单价格精度 |
| 买卖价差 | 判断 maker 是否有空间 |
| 成交速度 | 判断等待时间 |
| 队列数量 | 判断排队风险 |

## 10. theoreticalListingPrice

理论挂单目标价用于观察池，不代表即时成交。

```text
theoreticalListingPrice = 当前最低卖盘附近的可竞争挂单价
```

必须同时输出：

| 字段 | 含义 |
| --- | --- |
| listingFillProbability | 目标价成交概率 |
| queueScore | 排队竞争评分 |
| demandScore | 买盘需求评分 |
| spreadScore | 买卖价差评分 |

## 11. 两类机会消费协议

### 11.1 即时求购利差

| 字段 | 来源 |
| --- | --- |
| sellPrice | A 侧 `takerSellPrice` |
| buyPrice | B 侧 `spotBuyPrice` |
| quantity | min(库存, bid 深度, ask 深度, 余额) |
| liquidity | min(A 侧流动性, B 侧流动性) |
| executable | 只有利润、流动性、库存、余额都达标才为 true |

### 11.2 理论挂单利差

| 字段 | 来源 |
| --- | --- |
| sellPrice | A 侧 `theoreticalListingPrice` |
| buyPrice | B 侧 `spotBuyPrice` |
| quantity | min(库存, ask 深度, 余额) |
| liquidity | min(A 侧流动性, B 侧流动性, 挂单成交概率) |
| executable | 固定为 false |

## 12. 费用、毛利差与净利差

| 成本项 | 即时求购利差 | 理论挂单利差 |
| --- | --- | --- |
| 卖家费 | 扣除 | 扣除 |
| 卖出 taker 费 | 扣除 | 不适用 |
| 卖出 maker 费 | 不适用 | 扣除 |
| 买家费 | 扣除 | 扣除 |
| 买入 taker 费 | 扣除 | 扣除 |
| 充值成本 | 扣除 | 扣除 |
| 汇率成本 | 扣除 | 扣除 |
| 风险缓冲 | 较低 | 较高 |

输出口径必须分两层：

| 口径 | 公式 | 用途 |
| --- | --- | --- |
| 毛利润 | A 侧卖出毛额 - B 侧买入毛额 | 观察原始平台价差 |
| 毛利差 | 毛利润 / B 侧买入毛额 | 按原始价差排序和筛选 |
| 手续费估算 | 卖出费 + 买入费 + 充值成本 + 汇率成本 | 判断交易成本吞噬程度 |
| 净利润 | A 侧卖出净到账 - B 侧买入总成本 - 风险缓冲 | 判断是否具备采纳价值 |
| 净利差 | 净利润 / B 侧买入总成本 | 默认机会排序口径 |

## 13. 异常与剔除

| 异常 | 处理 |
| --- | --- |
| 单平台价格偏离 referencePrice 过大 | 降低置信度或剔除 |
| 数据来自缓存且过期 | 不得进入强执行 |
| bid 或 ask 深度不足 | 剔除即时机会 |
| B 侧 ask VWAP 后利润消失 | 剔除 |
| 充值成本未知 | 不得进入强执行 |
| 费率未知 | 不得进入强执行 |
| 库存不足 | 剔除 |
| 余额不足 | 剔除 |
| 挂单成交概率低 | 理论机会降级或剔除 |

## 14. 最小验收

| 验收项 | 标准 |
| --- | --- |
| 标准化盘口 | 能生成 bids、asks、freshness、tradeVelocity |
| 参考价 | 能生成多平台加权中位数 |
| bid VWAP | 能计算即时卖出可成交均价 |
| ask VWAP | 能计算现货买入可成交均价 |
| 理论挂单价 | 能输出目标挂单价和成交概率 |
| 两类机会 | 能分别输出即时求购和理论挂单 |
| 费用拆解 | 能显示毛利差、净利差、充值成本和风险缓冲 |
| 流动性过滤 | 不足时不得采纳 |
| Dry Run | 能按机会类型生成不同预案 |
